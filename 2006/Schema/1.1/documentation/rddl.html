Resource directory for "documentation" folder

The "documentation" folder contains files documenting the schema. They have been created either manually or (mostly) automatically based on the schema files. For more information on UBIF and SDD see the SDD Wiki, especially the Charter.

Note: While this document is human-readable, RDDL-aware processors may use it to automatically retrieve resources such as different schema versions for validating documents. See http://www.rddl.org for further information.

Resources

  • Two files document special schema-like files containing only base information and intellectually property rights and licences:
    • UBIF_(c)_Overview.html
    • SDD_(c)_Overview.html
  • The following files document the alternative entry level schema files (i.e. one of which will usually be referenced in instance documents):
    • UBIF_Overview.html
    • SDD_Overview.html
    • SDD_ForObjects_Overview.html
  • The entry level schema files are based on the following library files:
    • UBIF_BasePattern_Overview.html
    • UBIF_CoreOntology_Overview.html
    • UBIF_CoreExtensions_Overview.html
    • UBIF_LinneanCore_Overview.html
    • UBIF_TypeLib_Overview.html
    • SDD_TypeLib_Overview.html
  • A special kind of library file contain schema enumerated values (= controlled vocabularies). These are documented both in the overview format used for other schema files, and in a special format, giving details of the documentation value embedded in the schema:
    • UBIF_EnumLib_Overview.html
    • UBIF_EnumData.html
    • SDD_EnumLib_Overview.html
    • SDD_EnumData.html
  • Note for application builders/software designers: To simplify application building, enumerated values are also provided as UBIF/SDD data files (using DescriptiveConcept/ConceptStates):
    • UBIF_EnumData.xml and
    • SDD_EnumData.xml, which use a supplementary specification schema:
    • UBIF_EnumSpecs.xsd in a separate namespace.
    • (These files all come from a separate RDDL subdirectory)
